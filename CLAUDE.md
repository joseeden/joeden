# Claude Code Instructions

These instructions apply when Claude Code creates or updates files in this repository.

## Documentation Style

When creating or updating Markdown pages in `docs/`, follow `.agents/KB-Writing-Guide.md`.

Every KB page must include frontmatter.

Frontmatter should match nearby pages in the same folder.

Use a short and simplified introduction.

Use short sections and short paragraphs.

Use multiple short bullet items when they make the page easier to scan.

Capitalize the first word of every bullet item.

Use a table when a list is clearer as rows and columns.

Format Markdown tables with padded columns so the pipes align in the raw source file.

Use short inline `**Note**:` paragraphs for brief reminders or caveats.

Keep inline notes to one short paragraph.

Use Docusaurus admonitions when a paragraph or phrase is better shown as a callout.

Use the correct admonition type: `note`, `tip`, `info`, `warning`, or `danger`.

Use standard Docusaurus admonition syntax with matching opening and closing `:::` markers.

Avoid sub-bullets unless they are needed.

Do not use em dashes.

Do not use asyndetons, and use conjunctions when listing related ideas in prose.

Keep the tone practical, calm, and direct.
